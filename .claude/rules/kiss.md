# KISS (Keep It Simple, Stupid)

- Choose the simplest solution that works. If a 5-line function solves it, don't write a 50-line abstraction.
- No premature optimization. Profile first, optimize second.
- Prefer flat over nested. If you have 3+ levels of indentation, refactor.
- One file = one clear purpose. If you can't describe what a file does in one sentence, split it.
- Avoid clever code. Write for the next developer, not to impress.
- GDScript is intentionally simple — embrace it. Don't fight the language with design patterns it doesn't need.
